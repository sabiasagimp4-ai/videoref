## 実装計画（ウェーブ分割）

差別化価値・実装容易性・要望（UI美化）のバランスで4波に分割。各波は独立してリリース可能。

- **Wave 1 — 動画プレイヤー強化（A1〜A5）**：再生速度・フレームステップ・区間ループ・ホバースクラブ・フレーム書き出し。videoref の核。A1〜A4はフロントのみ、A5はffmpeg（既存利用）。
- **Wave 2 — UI磨き込み（D1〜D5）**：クイックルック・テーマ・サムネサイズ・UIズーム・バリュートグル。要望の「美しく」に直結。ほぼCSS/フロント。
- **Wave 3 — 取り込み/検索（B1, C1, A7, B3）**：クリップボード貼付け・グローバル検索・URLショートカット保存。
- **Wave 4 — 整理強化（C2, C3, B2）**：タググループ・知覚ハッシュ類似検出・ウォッチフォルダ。やや重い。

安定化（D6）は各波の実装中に並行して対応。

## Wave 1 詳細設計（最優先）

### フロント（public/app.js, index.html, style.css）
モーダル（`openVideoModal`）にプレイヤーバーを追加：
- 速度: `<select>` 0.25/0.5/1/1.5/2/4 → `video.playbackRate`
- フレームステップ: `← →` ボタン＋キー。`video.currentTime ± (1/fps)`。fps は file.meta から、無ければ 30 既定。
- 区間ループ: `IN`/`OUT` ボタンで現在時刻を記録、ループ中は `timeupdate` で OUT 到達時に IN へシーク。
- ホバースクラブ: タイムライン上の `mousemove` で `video.currentTime` を一時シーク（軽量版。サブ動画要素不要）。
- 現在フレーム保存: ボタン → `POST /api/frame`（fileId, time）。

### バック（server.js）
- `POST /api/frame { fileId, time }` → ffmpeg `-ss {time} -i {path} -frames:v 1` でPNGをライブラリ内 `_frames/` に書き出し、新規アイテムとして登録（既存 import パターン流用）。`jobLibraryPath` スナップショット必須。
- A5連番書き出しは `POST /api/frames/extract { fileId, interval }` で連番PNG。
- ライブラリパス・スナップショット規約を厳守（既存規約）。

### 受け入れ条件
分離テスト環境（temp dir + 非デフォルトポート + 明示env）で：再生速度切替・フレームステップ・区間ループが動作し、フレーム書き出しがライブラリに新規アイテムとして現れること。実データ/AppData には一切触れない。
